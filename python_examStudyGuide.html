Debugging Code

4.1: Debugging Techniques

Technique Definition
Breakpoints Stop program execution at a certain line to inspect the state of variables and control flow.
Step Over Execute the current line and skip over any function calls within it.
Step Into Enter the function and pause at the first line, allowing deeper inspection.
Watch Variables Monitor the value of variables during execution to track changes and identify problems.

4.2: Tracing Techniques

  • Print Statements: Insert print statements to display variable values and flow during execution.
  • Logging: Use the logging module to capture detailed information about the program's execution flow.
  • Traceback: Python provides traceback information when errors occur, which helps in locating the error in the code.

4.3: Example of Debugging Code

    def divide(a, b):
        print(f"Dividing {a} by {b}")  # Trace input
        if b == 0:
            print("Error: Division by zero!")
            return None
        return a / b
    
    result = divide(10, 0)  # Debugging step here
    print(result)

Functional Programming

6.1: Functions with and without Return Values

Type Definition Example
Function with Return Value A function that returns a result to the caller using the return keyword. 
def add(a, b): return a + b
Function without Return Value A function that performs an action but does not return any result. 
def greet(name): print(f"Hello, {name}!")

6.2: Functions in Control Structures

  • Using Functions in If Statements: Functions can return values that are evaluated in conditional statements.
  • Using Functions in Loops: Functions can be called repeatedly inside loops to perform an action or calculation.

6.3: Passing Arguments by Reference and Value

Passing Type Explanation Example
Passing by Value When immutable objects (like strings and integers) are passed, a copy is made, and changes do not affect the original object. 
def modify_string(s): s = "Modified" return s
Passing by Reference When mutable objects (like lists or dictionaries) are passed, the original object is modified. 
def modify_list(lst): lst.append(4)

6.4: Example of Passing by Reference and Value

    # Passing by value (string)
    original = "Original"
    modified = modify_string(original)
    print(original)  # Output: Original
    print(modified)  # Output: Modified
    
    # Passing by reference (list)
    my_list = [1, 2, 3]
    modify_list(my_list)
    print(my_list)  # Output: [1, 2, 3, 4]

Sequential File Processing

7.1: Definitions

Term Definition
Text File A file that contains human-readable characters, typically encoded as ASCII or UTF-8.
Binary File A file that contains data in a format that is not human-readable, often used for images, audio, and other media.
CSV (Comma Separated Values) A plain text file format that stores tabular data, with each line representing a row and each value separated by commas.

7.2: Methods for File Handling

  • Creating a File: To create a file, open it in write mode (`w`). If the file doesn't exist, Python will create it.
  • Reading a File: Use read (`r`) mode to open a file for reading. You can read the entire content or line by line.
  • Writing to a File: Use write (`w`) mode to write data to a file. It will overwrite the existing content.
  • Appending to a File: Open the file in append (`a`) mode to add content to the end without erasing existing data.

7.3: File Handling Example

Hereâ€™s an example of creating, writing, and reading a text file:

    # Creating and writing to a file
    with open("example.txt", "w") as file:
        file.write("Hello, World!\n")
        file.write("This is a text file.")
    
    # Reading from a file
    with open("example.txt", "r") as file:
        content = file.read()
        print(content)  # Output: "Hello, World!\nThis is a text file."

Writing Error-Free Code

8.1: Explain Exceptions and Error Handling

Term Definition
Exception An error that occurs during the execution of a program that interrupts its normal flow.
Error Handling The process of catching and managing exceptions to ensure that the program doesn't crash and provides useful feedback.

8.2: Write Code that Uses Try, Except, Else, and Finally Blocks to Trap and Handle Errors

  • try: Contains the code that might raise an exception.
  • except: Handles specific exceptions if they occur during the execution of the try block.
  • else: Executes if no exceptions were raised.
  • finally: Always executes, regardless of whether an exception occurred, useful for cleanup actions (like closing files).
    try:
        x = 10 / 0  # This will raise a ZeroDivisionError
    except ZeroDivisionError:
        print("Cannot divide by zero!")
    else:
        print("Division successful!")
    finally:
        print("This will always run, regardless of the error.")

8.3: Write Code to Raise Exceptions

You can raise exceptions manually using the raise keyword to enforce conditions in your program.

    def check_age(age):
        if age < 18:
            raise ValueError("Age must be 18 or older.")
        return age
    
    try:
        check_age(16)
    except ValueError as e:
        print(f"Error: {e}")

8.4: Write Assert Statements to Enforce Conditions

Assert statements are used to check if a condition is True. If it isn't, an AssertionError is raised.

    age = 20
    assert age >= 18, "Age must be 18 or older."  # This will pass as the condition is true
    
    age = 16
    assert age >= 18, "Age must be 18 or older."  # This will raise an AssertionError

Collections

9.1: List Operations

Lists are ordered collections that can hold multiple items, and you can modify the listâ€™s elements. Lists are defined with square brackets [ ].

Operation Definition Example
Appending Adding an item to the end of the list using append(). 
my_list = [1, 2, 3] 
my_list.append(4)  # Output: [1, 2, 3, 4]
Inserting Inserting an item at a specified position using insert(). 
my_list = [1, 2, 3] 
my_list.insert(1, 4)  # Output: [1, 4, 2, 3]
Removing Removing an item by value using remove() or by index using pop(). 
my_list = [1, 2, 3] 
my_list.remove(2)  # Output: [1, 3] 
my_list.pop(1)  # Output: 3, my_list: [1]
Sorting Sorting the list in ascending or descending order using sort(). 
my_list = [3, 1, 2] 
my_list.sort()  # Output: [1, 2, 3]
Reversing Reversing the order of elements using reverse(). 
my_list = [1, 2, 3] 
my_list.reverse()  # Output: [3, 2, 1]

9.2: Dictionary Operations

Dictionaries store key-value pairs, and you can retrieve, add, and modify data using keys. Dictionaries are defined with curly braces { }.

Operation Definition Example
Adding/Updating Adding or updating a key-value pair in a dictionary. 
my_dict = {"name": "Alice"} 
my_dict["age"] = 25  # Output: {"name": "Alice", "age": 25}
Accessing Accessing values using keys. 
my_dict = {"name": "Alice", "age": 25} 
name = my_dict["name"]  # Output: Alice
Removing Removing a key-value pair using pop() or del. 
my_dict = {"name": "Alice", "age": 25} 
my_dict.pop("age")  # Output: 25, my_dict: {"name": "Alice"}
Keys and Values Getting all keys or all values using keys() and values(). 
my_dict = {"name": "Alice", "age": 25} 
keys = my_dict.keys()  # Output: dict_keys(['name', 'age'])

9.3: Set Operations

Sets are unordered collections of unique items. They are defined using curly braces { }, and they do not allow duplicates.

Operation Definition Example
Adding Adding an element to a set using add(). 
my_set = {1, 2, 3} 
my_set.add(4)  # Output: {1, 2, 3, 4}
Removing Removing an element from a set using remove() or discard(). 
my_set = {1, 2, 3} 
my_set.remove(2)  # Output: {1, 3}
Set Operations Performing mathematical set operations such as union, intersection, and difference. 
set_a = {1, 2, 3} 
set_b = {3, 4, 5} 
union = set_a | set_b  # Output: {1, 2, 3, 4, 5}

9.4: Tuple Operations

Tuples are ordered collections, like lists, but they are immutable. Once created, you cannot change the items in a tuple. They are defined with parentheses ( ).

Operation Definition Example
Accessing Accessing items in a tuple by index. 
my_tuple = (1, 2, 3) 
item = my_tuple[1]  # Output: 2
Concatenation Concatenating two tuples using the + operator. 
my_tuple = (1, 2) 
new_tuple = my_tuple + (3, 4)  # Output: (1, 2, 3, 4)
Repetition Repeating a tuple using the * operator. 
my_tuple = (1, 2) 
repeated = my_tuple * 3  # Output: (1, 2, 1, 2, 1, 2)
